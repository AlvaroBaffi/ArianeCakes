# Documento de Requisitos do Site de Confeitaria

## 1. Introdução
Este documento apresenta os requisitos funcionais e não funcionais do site da confeitaria.  
As informações foram coletadas por meio de entrevista com a cliente e refletem as necessidades atuais e futuras do negócio.

---

## 2. Escopo do Projeto
O site será desenvolvido para apresentar os produtos da confeitaria, facilitar a comunicação com os clientes e servir como vitrine digital para atrair novos consumidores.  
Além disso, deverá ser de fácil manutenção e preparado para futura expansão, incluindo a possibilidade de loja virtual.

---

## 3. Requisitos Funcionais
Os requisitos funcionais descrevem o que o sistema deve fazer.

### 3.1. Sobre o Negócio
- Exibir a história da confeitaria e seu propósito.  
- Destacar os valores: qualidade, dedicação, zelo, respeito e empatia.  
- Evidenciar o público-alvo: mulheres jovens, principalmente mães.  
- Apresentar os produtos mais procurados (com foco nos copos da felicidade e morango do amor).  

### 3.2. Objetivos do Site
- Oferecer praticidade para o cliente na navegação e busca por informações.  
- Servir tanto para atrair novos clientes quanto para manter os atuais informados.  

### 3.3. Identidade Visual
- Utilizar o site "Yooga" como referência de design.  
- Transmitir elegância, alegria e aconchego, evitando simplicidade excessiva.  

### 3.4. Conteúdo do Site
- Informações essenciais: contato, cardápio, preços, horário de funcionamento.  
- Depoimentos de clientes (feedbacks).  
- Seção de promoções e novidades.  

### 3.5. Funcionalidades
- Catálogo de produtos com preços.  
- Integração com Instagram.  
- Mapa de localização (Google Maps).  

### 3.6. Manutenção e Futuro
- Cliente será responsável pela atualização do site após a entrega.  
- O site deve permitir modificações frequentes (novos produtos, fotos, promoções).  
- Possibilidade de expansão futura para loja virtual quando houver loja física.  

---

## 4. Requisitos Não Funcionais
Os requisitos não funcionais descrevem restrições, qualidades e características que o sistema deve atender.

### 4.1. Usabilidade
- O site deve ser intuitivo, com navegação clara e acessível.  
- Deve permitir que o cliente encontre informações em poucos cliques.  

### 4.2. Desempenho
- O carregamento das páginas deve ser rápido, mesmo em conexões móveis.  
- Suporte a múltiplos acessos simultâneos sem perda significativa de desempenho.  

### 4.3. Segurança
- O site deve ser protegido contra acessos não autorizados.  
- Uso de protocolo HTTPS para transmissão segura de dados.  

### 4.4. Manutenibilidade
- O site deve ser facilmente editável pela cliente (sem necessidade de conhecimento técnico avançado).  
- Estrutura modular que facilite futuras expansões.  

### 4.5. Compatibilidade
- O site deve ser responsivo, funcionando em dispositivos móveis, tablets e desktops.  
- Compatível com os principais navegadores (Chrome, Firefox, Safari, Edge).  

### 4.6. Escalabilidade
- O sistema deve permitir a futura adição de novas funcionalidades, como carrinho de compras e pagamentos online.  

---

## 5. Conclusão
Este documento apresenta os requisitos coletados com a cliente e servirá de guia para o desenvolvimento do site.  
Ele deverá ser revisado e atualizado conforme o andamento do projeto e novas necessidades surgirem.
